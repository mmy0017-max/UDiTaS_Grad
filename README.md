# UDiTaS_Grad

**CRISPR-Cas9 Structural Variant Detection Pipeline**

## Features
* **UMI-based Deduplication:** Accurately filters out PCR duplicates using Unique Molecular Identifiers (UMIs) to ensure accurate molecular counting.
* **Automated Alignment:** Integrates seamlessly with Bowtie2 for fast, local sequence alignment.
* **Target Detection:** Automatically detects the target chromosome based on initial mapping if not explicitly provided (`Auto` mode).
* **SV Classification:** Analyzes CIGAR strings to categorize reads into Wild Type (WT), Small Deletions, Large Deletions, Insertions, Substitutions, Translocations, and Inversions.
* **Visualization:** Generates figures, including a pie chart of mutation frequencies and a schematic of alignment patterns around the predicted cleavage site.

## Prerequisites
Ensure you have the following installed on your system:
* **Anaconda 3.13.9**
* **Biopython**
* **Bowtie2:** Must be installed and accessible in your system PATH variable. (linux & MacOS only)

## Configuration
The pipeline currently utilizes hardcoded variables for configuration. Before running the script, open `UDiTaS_Grad.py` and update the variables under the `### Configuration ###` section:

1. **Input Files:**
   * `FASTQ_UMI_FILE`: Path to your UMI reads (e.g., R2).
   * `FASTQ_TARGET_FILE`: Path to your target reads (e.g., R3).
   * *Note: The script natively supports both uncompressed `.fastq` and gzipped `.fastq.gz` files.*

2. **Genome Index:**
   * `GENOME_INDEX`: Path to the Bowtie2 reference genome index (e.g., `GRCh38_clean/hg38_clean`).

3. **Sequences:**
   * `PRIMER_SEQ`: The exact primer sequence used in your library preparation.
   * `ORIGIN_SEQ`: The reference sequence adjacent to the primer (leave as `""` if not applicable).

4. **Settings:**
   * `TARGET_CHR`: Set to a specific chromosome (e.g., `"chr2"`) or leave as `"Auto"` to let the script detect the primary mapping target.
   * `LARGE_DELETION_THRESHOLD`: Size threshold (in bp) to classify a deletion or soft-clip as a "Large Deletion" (default: `50`).
   * `LARGE_DELETION_THRESHOLD`: Size threshold (in bp) to classify a deletion as a "Large Deletion" (default: `50`).
   * `SOFT_CLIP_BUFFER`: Size threshold (in bp) to classify a soft-clip added to above value (default: `5`).
   * `BOWTIE2_THREADS`: Number of CPU cores to allocate for Bowtie2 alignment (default: `8`).

## Usage
Once the configuration variables are set, execute the script from your terminal:
```bash
python UDiTaS_Grad.py
```

## Output Files
The script creates an `uditas_output` directory containing the following generated files:
* `trimmed.fastq`: The processed reads after UMI extraction and primer/origin sequence trimming.
* `trimmed.sam`: The raw alignment output generated by Bowtie2.
* `align_summary.tsv`: A detailed TSV file containing read headers, CIGAR strings, and mutation classifications for every unique molecule.
* `mutation_visualization.png`: A visual summary of the run, featuring a pie chart of mutation categories and a visual alignment map highlighting substitutions, indels, and the predicted cleavage position.
